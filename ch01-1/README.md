# 클래스와 객체

## ● 클래스
- 객체를 만들어내기 위해 존재하는 기본적인 틀이며, 변수와 함수를 포함하여 대상의 속성과 행위를 정의한다. (설계도)

## ● 객체
- 클래스에서 생겨나 실체화가 된 것.

# 절차지향과 객체지향 프로그래밍의 차이점

## ● 절차지향 프로그래밍
- 기능적인 절차에 따라 순서를 나열하고, 그 흐름에 따라 프로그램을 작성하는 방식 

## ● 객체지향 프로그래밍
- 객체들을 정의한 후, 각 개체들의 관계에 따른 상호 작용을 통해 구현하는 방식
  
  
  
# 라이브러리
- 프로그래밍을 할 때 자주 반복적으로 사용되는 특정 기능들 (ex: printf, scanf 등)을 편하게 이용하기 위해 사전에 작성해 놓은 코드들의 집합.
  
  
  
# 프로그램의 개발 과정

## ● 편집 단계 
- C언어로 컴퓨터가 처리할 소스 코드를 작성하는 단계, 보통 VS, VSCODE 등의 통합 개발 환경에서 진행된다. (C++의 확장자는 cpp이다)

## ● 컴파일 단계
- 소스 코드의 문법을 체크하고, 기계가 이해할 수 있는 저급 언어로 변환하는 단계. 컴파일러의 컴파일 후에는 목적 파일(object file) 이 산출된다.

## ● 링크 단계
- 산출된 목적 파일은 단독으로 실행할 수 없는데, 프로그래밍 과정에서 사용한 표준 라이브러리의 기능들을 표준 라이브러리 파일과 연결해주는 링킹 과정이 필요하기 때문이다. 링킹 과정을 수행하는 프로그램을 링커(linker) 라 칭하며, 이 과정을 통해 산출된 최종 파일이 실행 파일(executable file) 이 된다.

## ● 실행 단계
- 실행 파일의 내용을 컴퓨터가 직접 수행하는 단계로, 로더(loader)를 이용해 실행 파일을 주기억장치(RAM) 에 저장한다.
