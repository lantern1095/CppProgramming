# 실습과제 1

## C++에서 생성자를 통하여 멤버변수의 초기화를 하는 이유를 설명하라.
- 클래스형 변수 (객체)가 만들어지는 순간과 동시에 초기화가 되기 때문에, 멤버변수에 쓰레기값이 들어가는 것을 방지해주고, 생성자 내부에서 잘못된 값에 대한 처리가 가능하여 (예시 : 음수를 입력받았으면 양수로 변환) 프로그래밍의 효율을 높여준다.

## 생성자 작성시 생성자 몸체에서 멤버를 초기화하는 것 보다 멤버 초기화 리스트에서 초기화하는 이유를 설명하라. 
- 몸체에서 초기화를 하면 객체를 생성한 다음 초기화를 하지만, 멤버 초기화 리스트에서 초기화를 한다면 객체를 생성하는 동시에 초기화가 이루어지기 때문에 더 효율적이기 때문이다.

## 위임, 타겟 생성자로 나누어 작성하는 이유를 설명하라
- 복수의 생성자를 만드는 과정에서, 같은 내용의 초기화를 반복한다면 코드 중복이 발생하고 코딩의 효율이 떨어지기 때문에, 위임 생성자는 타겟 생성자를 호출하여 초기화를 함으로서 코드 중복을 방지하고 코드 유지 및 보수에 있어서 효율을 높일 수 있다.

## 함수 몸체(body)에서 생성자를 직접 호출하면 어떻게 되는지 조사해보라
- 임시 객체만이 생성되고 소멸되어 결국 원래 초기화를 하려 했던 객체에는 아무 값도 들어가지 않아 쓰레기값이 된다.
