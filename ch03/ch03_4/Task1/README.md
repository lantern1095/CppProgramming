# 실습과제 1

## 접근지정자가 왜 필요한가?
- 외부에서 클래스 멤버로의 직접적인 접근을 차단함으로서 원치 않는 접근을 막아 보안성을 높이고, 실수로 값이 변해버리는 것을 방지하기 위해서이다.

## 생성자와 setter 멤버함수는 모두 멤버변수를 설정하는 기능이다. 차이점을 설명하라.
- 생성자는 객체 생성시의 초기화를 위한 용도이고, setter 멤버함수는 객체 생성 이후 멤버변수의 값을 변경할때 사용한다.

## 객체지향형 언어의 특징인 캡슐화와 정보은닉을 설명하라.
- 캡슐화 : 여러 가지의 데이터와 함수를 하나의 캡슐로 묶어 코드의 재사용성을 높임과 동시에 유지보수에 용이하도록 하는 것, C++에서는 클래스로 묶어 데이터를 보호한다.
- 정보 은닉 : 접근 지정자를 private / public 으로 설정함으로서 데이터를 비공개 / 공개 상태로 만들고, 이를 통해 외부로부터 클래스 내부의 요소들로의 접근을 제어하는 것이다.
